# CLAUDE.md — 黙走会 / minamo.run

## このプロジェクトについて

「誰も運営しないランニングコミュニティ」の型(フォーマット)を作るプロジェクト。
毎日走る人は毎日、週に一度の人は週に一度。走った日に、走った証を一枚のページに静かに残すだけ。
主催者は存在せず、集合場所も集合時刻もない。連絡手段が存在しないため、勧誘や営業が構造的に成立しない。

自分の積み重ねを「よかったな」と感じられて、誰かの積み重ねに会釈をして、そして会釈をされる。
それ以上のことは、何も起きない場所である。

オーナー(純さん)は「関わりを最小限にしながら、繋がりをエンパワーする」ことを目指している。
このプロジェクトは、giverであること・質の良い問いを立てること・青写真を持つこと、という
オーナーの人生の指針から生まれた。プロダクトはその思想の翻訳である。

同フォルダの PERSONA.md(藤井・小川・西田とアンチペルソナ)を必ず併読すること。
機能判断はPERSONA.md末尾の「4つの問い」で行う。

## 最重要原則(これに反する実装・提案はしない)

1. **足すより削る。** 迷ったら必ず削る側に倒す
2. **他者から見える、個人に紐づく数字を持たない。** フォロワー数・いいね数・ランキング・
   公開される個人スコアは思想への裏切り。自分にだけ見える記録(層1)はこの限りではない
3. **連絡手段を作らない。** DM・チャット・コメント・返信機能は永久に実装しない
4. **運用タスクをゼロにする。** 定期実行やリセット処理すら作らない(日付キーの設計で解決する)
5. **積み重ねは「累計」であって「連続」ではない。** ストリーク・連続日数・途切れの概念は永久に実装しない。
   休んでも減らない。責めない
6. **記録の入力は永久に任意。** 走った印だけで参加は完全に成立する。データがなくても二級参加者にならない
7. **全員が同一機能。** 有償機能・プレミアム・機能差は永久に作らない
8. **型は手放す。** MITライセンスで公開し、誰でも複製・改変・自主開催してよいことを明記する

## 憲章(叩き台 v0.2 — オーナーが今後さらに削る予定)

1. いつ走るかは、自分で決める。毎日でも、週に一度でも。集合場所も、集合時刻もない
2. 距離もペースも自由。歩いてもいい。休んでもいい。やめてもいい
3. 会話はしない。走った証は、ボタンひとつ、または #黙走会 の一投稿のみ
4. 自己紹介は不要。名乗らなくていい。誰であるかより、走ったかどうか
5. 勧誘・宣伝・営業のための接触は存在しない。この会には連絡手段がないため
6. 主催者はいない。参加者だけがいる
7. この型は誰のものでもない。誰でも、どこでも、許可なく開催してよい

## プロダクト仕様 — 三層の痕跡

### 層1:わたしの積み重ね(自分にだけ見える)
- 走った印(ボタン)= 参加の完全形。押すと「これで n 日目です」とだけ表示する
- **添え書き(任意):** 印を押した直後に一度だけ「添え書きしますか?」と静かに提示。
  距離(km)と時間(分)を記録できる。スキップ自由。あとから追記・修正も可能
- 添え書きの項目は距離と時間の2つのみに固定する。
  一筆(自由記述)は将来の追加機能候補として「保留箱」に保管し、現段階では実装しない
- 添え書きがある場合のみ、自分の累計距離を自分にだけ表示してよい
- **累計のみ。** 連続日数・ストリーク・「途切れました」表示は実装禁止(最重要原則5)
- データは localStorage のみで完結。サーバー送信・公開・共有・比較は永久に禁止
- 端末に宿る私的な御朱印帳という位置づけ。印は1日1回まで(同日の重複押下は加算しない)

### 層1への入力導線(初心者とアプリ利用者の両方をカバーする)

**A. 手入力(初心者・アプリ非利用者向け / Phase 1)**
- 手入力しやすさを最優先にUIを設計する:
  - 項目は距離と時間の2つ以内。必須項目ゼロ
  - `inputmode="decimal"` で数字キーパッドを直接起動。大きなタップ領域、片手操作で完結
  - 走り終えた直後の、汗ばんだ手での操作を想定する(小さなリンク・細かい選択肢を置かない)
  - 迷いを生む単位選択やフォーム階層は作らない。kmと分に固定

**B. ファイル取り込み(Strava / Garmin Connect 利用者向け / Phase 1〜2)**
- GPX / TCX / FIT ファイルのドラッグ&ドロップ(モバイルはファイル選択)に対応
- 解析はブラウザ内(クライアントサイド)で完結させ、ファイルをサーバーに送らない
- 距離・時間のみを抽出して層1に反映する。GPSルート・心拍等は読み捨てて保存しない

**C. iOSショートカット連携(Appleヘルスケア利用者向け / Phase 1〜2)**
- HealthKitはネイティブアプリ専用のため、Webから直接は読めない。代わりに
  「ヘルスケアの最新ワークアウトを読み、URLフラグメント(#d=5.2&t=31)でページを開く」
  ショートカットレシピを公開する
- フラグメントはサーバーに到達しないため、記録データがネットワークに出ない設計になる
- ページ側はフラグメントを読み取って添え書きに反映し、即座にURLから消去する

**D. 公式API連携(Strava / Garmin OAuth / Phase 3で判断・当面保留)**
- 「ログインなし」「アカウントなし」原則と衝突するため、Phase 3で改めて要否を判断する
- 導入する場合も:任意・読み取りのみ・最小権限。トークンをサーバーで預かる設計は避ける方向で検討
- 実装前に各APIの最新規約(レート制限・表示要件・利用制限)の確認を必須とする

**共通ルール:** いずれの導線でも、データの行き先は層1のみ。他者に見えるのは常に1ビット(走ったかどうか)だけ。

### 層2:今日の静かなつながり(誰のものでもない数字)
- ボタン押下で日付キー(JST)のカウンターを +1 し、「今日、共に走った人:n」を表示
- 個人には一切紐づかない。距離・時間などの添え書きデータはここに一切流れない
- **リセット処理は実装しない。** KVキーに日付(YYYY-MM-DD、JST基準)を含めることで、
  日が変われば新しいキーが使われるだけの設計にする

### 層3:会釈(1ビットの目撃)
- 走った印と同時に、その日限りの匿名IDを発行
- 今日走った他の参加者が、名前のない小さな光の点として並んで見える
- そのうち一点にだけ、一日に一度だけ触れられる(=会釈)
- 受け取った側は次にページを開いたとき「今日のあなたの走りに、誰かが会釈しました」の一文だけが灯る
- **仕様の核:**
  - 誰からの会釈かは分からない(返せない=互酬圧力ゼロの一方向の贈与)
  - 受信は「あった/なかった」の1ビットのみ。**数は表示しない**
  - 履歴は残らない。翌日にはすべて消える
  - 会釈を稼ぐ・競うことが構造的に不可能であること
  - 点には距離・時間・記録の情報を一切持たせない(速い人の点も、歩いた人の点も、同じ一粒の光)

### 共通
- 認証なし、アカウントなし、ログインなし
- トラッキング・分析タグ・Cookieによる追跡なし
- プッシュ通知・リマインドなし(走る動機はページの外にある)

## 技術構成

- **フロントエンド:** 静的HTML一枚。フレームワーク不使用。JSは最小限のvanilla
  (GPX/TCX/FIT解析のみ、軽量なパーサライブラリの利用を許可する)
- **バックエンド:** Cloudflare Workers + KV(無料枠内で完結させる)
- **ホスティング:** Cloudflare Pages または GitHub Pages
- **ドメイン:** minamo.run(取得予定。命名の経緯・由来は NAMING.md。日本語名「黙走会」は不変)
- **ライセンス:** MIT
- **README:** 「このページごと複製して、あなたの街の黙走会にしてよい」と明記する

## デザイン方針(4案からの一滴ずつの統合)

### ベース:和紙の佇まい
- 生成りの紙のような質感を背景にする(重い写真素材は使わず、CSSのテクスチャ/ノイズ/グラデーションで表現)
- 基調色:生成り(#f5f2ea 系)/墨色(#1a1a1a 系)。アクセントは朝焼けの淡い橙をごく控えめに
- 書体:明朝体(游明朝 / Noto Serif JP 等)。見出しも本文も静かに組む
- 余白を最大の装飾とする。常時のアニメーションなし

### Design A から一滴:御朱印の所作
- ボタンは「押す」ではなく「印を残す」行為として設計する。ラベルは「走った印を押す」
- 押せるのは1日1回。押した後、ボタンは静かに役目を終えた状態になる(連打・取消の概念を持たない)
- 添え書きの入力は「御朱印帳に日付と一筆を添える」所作として、印の直後に静かに差し出す
- 一回性と儀式性を、装飾ではなく挙動で表現する

### Design B から一滴:一粒の光
- 今日の参加者の気配は、名前のない小さな光の点の集まりとして描く
- 会釈は「一点の光にそっと触れる」操作。受け取った会釈は一粒の灯りとして表示する
- 点に個人情報・プロフィール・記録・リンクは一切持たせない

### Design C から一滴:走る喜びの一拍
- 走った印を押した瞬間にだけ、画面にほんのり夜明け色が差す約1秒の明転を入れる
- 高揚はこの一拍のみ。常時のビビッドカラー・スポーティな意匠・STARTモチーフは採用しない

### コピーの基調
- 「つながるために、増やさない」「比較ではなく、気配」「返せない会釈」
- タグライン:「走ったことは、誰かの気配になる。」
- ページ下部に「連絡手段なし・ランキングなし・ログインなし・広告なし・課金なし」を静かに明記する
  (アンチペルソナへの「ここに餌はない」という宣言を兼ねる)

### 画面構成(1ページ完結)
- ファーストビュー:黙走会の一文(「各自の場所で、黙って走る。主催者はいない。」)+ 走った印を押すボタン
- 印の直後:添え書き(任意)の静かな提示 → スキップまたは2項目入力
- 三層の表示:01 わたしの積み重ね / 02 今日の静かなつながり / 03 会釈
- 取り込み導線:「記録ファイルを添える」(ドラッグ&ドロップ / ファイル選択)を添え書きの隣に控えめに置く
- 憲章全文
- フッター:原則の明記と、複製自由(第7条)の案内

## Claude Code への働き方の指示

- 各ステップ完了時に「削れるものはないか」を一度レビューしてから次へ進む
- 機能追加の提案をする場合は、必ず「最重要原則」とPERSONA.mdの「4つの問い」で自己確認する
- 手入力UIは「走り終えた直後・屋外・片手・汗ばんだ手」を前提にレビューする
- ファイル取り込み実装時、解析がクライアントサイドで完結していること
  (ネットワークタブにファイル送信が現れないこと)を必ず確認する
- 公式API連携(Phase 3)に着手する場合は、実装前にStrava / Garmin各APIの最新規約を確認し、
  オーナーに要否の最終判断を仰ぐ
- abuse対策・セキュリティは最小限でよい(日次で消える匿名データに過剰な防御を足さない。
  簡易なレート制限程度で十分)
- 大きな設計変更はオーナーに提案ベースで確認する。オーナーの意思決定を尊重する
- 唯一の成功指標は「オーナーが今日も走りたいと思えているか」。数値目標は存在しない
