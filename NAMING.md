## ✅ 決定(2026-07-08)

**minamo.run(水面)に決定。**

- 選定理由: 発音容易(mi-na-mo)・目立ったブランド衝突なし・ロゴ意匠(水平線マクロン+一粒の月)との適合は候補中随一
- 黙走会との橋: 黙走は行為、みなもはその積み重ねが映る風景(粒が集まって水面になる。会釈カードの光の点フィールド=夜の水面)。
  「みなも」=「皆も」の読み替え(今日も、皆も、黙って走っている)

  moku / nagi / awai / shijima / kehai / kinari / shirushi / sotto / asayake / akane / shinonome / tomoru / yohaku / ashiato は空きを確認
- 次点は kehai.run(気配)だった(タグラインの核心語・衝突は小規模な掛け時計コレクションのみ)
- **残タスク(オーナー):** レジストラで minamo.run の価格確認(一般語のためプレミアム価格の可能性)と取得

## プロジェクトの世界観(名前が満たすべき前提)

- 誰も運営しないランニングコミュニティの「型」
- 走った日に、一枚のページに証を残すだけ。連絡手段は存在しない
- 主催者不在、勧誘不能、アカウントなし、ランキングなし、有償機能なし
- デザイン:和紙の佇まい、生成り+墨色、明朝体、余白が最大の装飾
- 積み重ねは「風景の複利」として可視化(粒→稜線→月→水面→夜明け)
- 思想:「足すより削る」「関わりは最小限、繋がりはエンパワー」
- 憲章第7条:「この型は誰のものでもない。誰でも、どこでも、許可なく開催してよい」

詳細は同フォルダの `CLAUDE.md` と `PERSONA.md` を参照。

## 名前が満たすべき5つの条件

1. **世界観との整合:** 静けさ・削ぎ落とし・非集合・非勧誘の思想を裏切らない
2. **宗教色ゼロ:** 禅・仏・道など、特定の宗教/宗派を連想させる語は綴りからも避ける
3. **不快な部分文字列を含まない:** 綴りの中に、日本語話者が見て違和感を持つ並びを含まない
4. **国際的な発音可能性:** 海外の話者が概ね発音できる
5. **ドメイン取得可能性:** `.run` TLDでのドメイン取得が現実的であること
   (レジストラでの空き確認は名前決定前に必ず実施する)

## これまでの検討経緯(要約)

### 却下された案と理由

- **mokuso.run** — 綴りに「kuso」が含まれることをオーナーが忌避。**採用しない**
- **mokusou / mokusoh** — 上と同じ理由に加え、綴りが野暮ったい。**採用しない**
- **zenso.run(禅走)** — 響きと切れ味は良いが、「zen」の3文字が綴りに残り、
  国内では宗教色、海外ではウェルネス系のクリシェとして読まれる。オーナーが宗教色を忌避。**採用しない**

### ロゴ意匠のアイデア(名前に依らず継承したい)

- 長音記号(マクロン)を水平線として延ばし、その上に一粒の月/光を置く意匠
- 母音の上に伸ばした線が、風景の複利の水平線と繋がる
- どの最終名でも、母音がひとつあればこの意匠は移植可能

## 現時点の残存候補

### 本命候補A: sumi.run(墨)
- **意味:** 墨。水墨画の墨
- **世界観の一貫性:**
  - 層1の風景は水墨画そのもの
  - 「筆が走る」という日本語の言い回し=墨と走ることは元来親戚
  - 一走=一筆、積み重ね=一枚の絵
  - デザインの基調色「墨色」と一致し、プロダクト名・思想・見た目が一本の墨線で貫かれる
- **国際性:** 「sumi-e」で世界的に知られる。発音容易
- **懸念:** 「sumi」は姓・地名としても存在。既存ブランドの調査必要

### 本命候補B: shizuku.run(雫)
- **意味:** 雫、しずく
- **世界観の一貫性:**
  - 層1の光の粒を「雫」と呼び直す詩性
  - 一走ごとに一雫が落ち、積み重なると水面になる
  - 節目240日の「水面が光を映す」と直接呼応
- **国際性:** 3音節でやや長いが、発音は容易
- **懸念:** 7文字でやや長い

### 控え候補C: moku.run(黙)
- **意味:** ドメインのドット位置で「黙.走」を訳し切る構造
- **世界観の一貫性:** 「黙走」の直訳。4文字で削る哲学の極点
- **懸念:**
  - 既存ブランド(計測機器のMokuなど)があり、識別性でやや弱い可能性
  - 4文字の.runドメインは既取得の可能性が高い

### 控え候補D: awai.run(あわい/間・淡い)
- **意味:** 古語で「物と物のあいだ」。触れ合わずに繋がる本プロダクトの核心
- **世界観の一貫性:** 「淡いつながり」とも読める二重の意味
- **懸念:** 日本語話者以外には意味が伝わりにくい

### 控え候補E: shijima.run(しじま)
- **意味:** 夜のしじま。静寂
- **世界観の一貫性:** 詩性は候補中随一
- **懸念:** 意味を知らない日本語話者も多く、認知獲得のコスト高

## Claude Codeでの検討タスク

### Step 1: 追加候補のブレスト
上記5つの条件を満たす追加候補を、和語・自然物・日本文化の語彙から広く生成する。
参考にしてよい語彙領域:自然現象、和紙の道具、書の用語、ランニング動作の擬態語、
古語、季語(ただし派手すぎない静かな語のみ)。

### Step 2: 各候補のスクリーニング
候補ごとに以下をチェック:
- [ ] 綴りに不快な部分文字列(kuso等)を含まない
- [ ] 宗教/宗派を直接連想させる語(zen, dharma, karma, tao等)を含まない
- [ ] 日本語話者・非日本語話者いずれも発音可能
- [ ] ネガティブな二次連想(スラング等)がない(英語・主要言語で検索)
- [ ] `.run` ドメインの空き状況(WHOIS等で確認)
- [ ] 同名の既存ブランド・OSSプロジェクトの有無(GitHub / Producthunt / 商標データベースで検索)

### Step 3: 3〜5案に絞り込みオーナーに提示
残った候補を、以下の観点で1案ずつ短評をつけて提示:
- 世界観との一貫性(どの節・どのモチーフと呼応するか)
- 発音・視認性
- ロゴ意匠(水平線マクロン)の適合度
- 主要な懸念点

### Step 4: 決定後の一括置換
オーナーが最終決定したら、以下を一括で更新:

**プロジェクト内ファイル:**
- `CLAUDE.md`:文中の「mokuso」「mokuso.run」をすべて置換
- `PERSONA.md`:同上
- `README.md`:同上(存在すれば)
- ソースコード内のドメイン参照
- 憲章文中の表記

**プロジェクト外(オーナーが実施):**
- ドメイン取得(`.run`)
- GitHubリポジトリ名
- SNSハンドル(必要であれば)

## 表記ルール(名前が決まった後に適用)

- **日常表記:** すべて小文字(例:sumi、sumi.run)。ドメイン・ハンドル・リポジトリ・文中で統一
- **式典表記:** 大文字ワードマーク。表紙や刊行物など特別な場面のみ
- **ロゴ意匠:** 母音の上にマクロン=水平線を延ばし、右端に一粒の光/月を置く
- **日本語コミュニティ名:** 「黙走会」で不変。アルファベット名と併記する場面もあり得る

このルールも決定後、`CLAUDE.md`の「ブランド表記」節として正式に追記する。
