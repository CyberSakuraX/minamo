## 1. ゴール

「誰も運営しないランニングコミュニティ」の型を、静的HTML1枚 + Cloudflare Workers + KV で実装し、
無料サブドメイン上に公開する。初回の一日は **2026-07-12（日）朝** を予定。

唯一の成功指標は「オーナーが今日も走りたいと思えているか」。数値目標は存在しない。

## 2. 前提（このセッションで確定済みの意思決定）

| 項目 | 決定 |
|---|---|
| カラートーン | **暁（akatsuki）1本に絞る。切替UIは実装しない**（[03_デザイントークン.md](03_デザイントークン.md)） |
| バックエンド | Cloudflare Workers + KV（確定） |
| ホスティング | **Step 4 で判断**。Cloudflare Pages / GitHub Pages 両対応の手順を [05_デプロイ手順.md](05_デプロイ手順.md) に用意済み |

| デザインの正 | `design-reference/MokusoApp.dc.html`（Claude Design プロトタイプの復元原本・SHA-256検証済み） |

## 3. リポジトリ構成（実装後の姿）

```
mokusoh_run/
├── CLAUDE.md                      # 思想・原則（既存・全実装判断の憲法）
├── PERSONA.md                     # ペルソナと4つの問い（既存）
├── LICENSE                        # MIT（Step 4）
├── README.md                      # 複製自由の明記・フォーク手順（Step 4）
├── .gitignore                     # node_modules / .wrangler / .dev.vars
├── public/                        # ★静的ページ。デプロイ対象ディレクトリ
│   ├── index.html                 # 1枚完結（CSS/JSインライン）
│   └── js/
│       └── fit-reader.js          # FIT最小パーサ（動的import。Step 3）
├── worker/                        # ★Cloudflare Workers
│   ├── wrangler.toml
│   └── src/index.js               # 全API（1ファイル）
├── docs/                          # 本ドキュメント群
└── design-reference/              # デザイン原本（改変しない）
```

**index.html を1ファイル完結にする理由:** 憲章7条「この型は誰のものでもない。誰でも、どこでも、
許可なく開催してよい」を、「コピーすべきファイルが実質1つ」という形で技術的に成立させるため。
唯一の例外は FIT パーサ（バイナリ解析でコード量が多く、FITを使わない訪問者に読ませないため動的importで分離）。

## 4. 実装ステップ

各ステップ完了時に「**削れるものはないか**」を一度レビューしてから次へ進むこと（CLAUDE.md の働き方指示）。

### Step 0 — 完了済み（このセッション）
- デザインプロトタイプの全ソース抽出・復元（design-reference/、SHA-256一致確認済み）
- 計画書・設計図一式（docs/00〜06）

### Step 1（90分）— リポジトリ骨格 + 静的ページ（層1完結）

**作業項目:**
1. `git init`、`.gitignore` 作成（node_modules / .wrangler / .dev.vars）
2. `public/index.html` を新規作成:
   - [03_デザイントークン.md](03_デザイントークン.md) のCSS変数・フォント読み込み
   - [01_設計図_画面と挙動.md](01_設計図_画面と挙動.md) の全セクション（**憲章全文・フッター含む** — プロトタイプ未実装分）
   - 層1のJS: 押印 / 明転 / 印影（漢数字落款）/ 添え書き手入力 / 書き直し / 累計表示
   - `API_BASE = ""` の状態で層2・3が静かに畳まれるデグラデーション動作
3. 文言・寸法（180px印 / 46px印影 / 188px点フィールド / rotate角 / グラデーション）は
   design-reference から**忠実に**移植する（勝手な意訳・改変をしない）

**受け入れ基準:**
- [ ] DevToolsオフライン状態で全機能（押印 → n日目 → 添え書き → 累計）が動く
- [ ] 同日2回目の押印が加算されない。localStorage を消すと「まだ、印はありません。」に戻る
- [ ] ページ内のどこにも「連続」「ストリーク」「途切れ」に相当する表示・変数名がない（grepで確認）
- [ ] 手入力UI: 必須項目ゼロ / `inputmode="decimal"` / タップ領域44px以上（「走り終えた直後・屋外・片手・汗ばんだ手」でレビュー）

**削るレビュー観点:** 見出しの装飾を減らせないか / 文言をプロトタイプより増やしていないか /
添え書きフォームの要素数（input 2 + ボタン2 + 案内文 + ファイル導線）で打ち止めか

### Step 2（90分）— Cloudflare Workers + KV（層2 → 層3の順）

**作業項目:**
1. `worker/` 作成、wrangler.toml（[05_デプロイ手順.md](05_デプロイ手順.md) 参照）、`wrangler kv namespace create MINAMO_KV`
2. `src/index.js` に [02_設計図_API_データ.md](02_設計図_API_データ.md) の4エンドポイント + CORS + メモリキャッシュ + 簡易レート制限
3. `wrangler deploy` で workers.dev に公開。curl で動作確認
4. index.html の `API_BASE` を設定し、層2・3のクライアント処理を接続

**受け入れ基準:**
- [ ] curl で stamp → today → bow → bow確認 の一巡が仕様通りのJSONを返す
- [ ] `wrangler kv key list` で全キーに expiration が付いている
- [ ] ブラウザ2窓（通常 + シークレット）で会釈の送受信が成立し、**受信側に数がどこにも出ない**
- [ ] Worker コードを grep して `CF-Connecting-IP` が put / console.log に渡っていない
- [ ] API_BASE を偽URLにすると 02 が「—」・03 が非表示になり、層1は無傷

**削るレビュー観点:** レスポンスに余計なフィールド（バージョン・時刻等）を足していないか /
エラーレスポンスが饒舌になっていないか

### Step 3（60分）— ファイル取り込み + フラグメント受け口

**作業項目:** GPX/TCX のインラインパーサ、`public/js/fit-reader.js`（自作最小 or MITライブラリのvendoring）、
input[type=file] + ドラッグ&ドロップ、`#d=5.2&t=31` の読み取り→添え書きプリフィル→URL即消去
（仕様詳細は [01_設計図_画面と挙動.md](01_設計図_画面と挙動.md) §7 と [02_設計図_API_データ.md](02_設計図_API_データ.md) §8）

**受け入れ基準:**
- [ ] サンプル GPX / TCX / FIT から km・分が入力欄に入る（自動保存はされない）
- [ ] **取り込み操作中、DevTools Network タブに新規リクエストが1件も出ない**（fit-reader.js の同一オリジン初回GETのみ許容）
- [ ] オフライン状態でも取り込みが動く
- [ ] `#d=5.2&t=31` 付きで開くと欄が埋まり、URLから即消える

**削るレビュー観点:** 抽出フィールドが距離・時間の2つを超えていないか /
FITパーサに使わないメッセージ型の処理が残っていないか / エラー文言が1種類で足りているか

### Step 4（30分）— ライセンス・README・公開・デプロイ

**作業項目:**
1. LICENSE（MIT）
2. README.md: プロジェクトの一文 / **「このページごと複製して、あなたの街の黙走会にしてよい」** /
   フォーク手順（**public/ フォルダごとコピー**し、API_BASE を空にするか自分のWorkerを立てる。
   Step 3 以降 js/fit-reader.js があるため1枚コピーでは不完全 — 2026-07-09 更新）/ docs への索引
3. GitHub リポジトリ公開（public）
4. **ホスティングをここで決定** → [05_デプロイ手順.md](05_デプロイ手順.md) の該当手順でデプロイ
5. Worker の `ALLOWED_ORIGINS` を本番URLに更新して再デプロイ

**受け入れ基準:**
- [ ] 公開URLでフル動作（層1〜3）
- [ ] README だけ読んだ第三者が複製できる
- [ ] リポジトリに秘密情報ゼロ（wrangler.toml のaccount_idは公開可、トークン類は絶対に含めない）

**削るレビュー観点:** README が1画面に収まるか / docs に保留箱の機能（一筆・公式API連携）への言及を増やしていないか

### Step 5（後日・小タスク）— iOSショートカットレシピ

docs/ios-shortcut.md を新規作成: 「ヘルスケアの最新ワークアウトを読み、`https://<公開URL>/#d=◯&t=◯` を開く」
レシピの作成手順とスクリーンショット。ページ側の受け口は Step 3 で完成済み。

**受け入れ基準:** 実機でショートカット実行 → 添え書きが埋まる → URLにデータが残らない。

### Step 6 — 最初の一日（2026-07-12 日曜朝 予定）

前夜チェック（デプロイ確認・KVが空・API疎通）→ オーナーが走る → 印を押す →「今日、共に走った人 1 人」が灯る。
以後、運用作業は存在しない（誰も何もしない）。
