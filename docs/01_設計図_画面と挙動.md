デザインの正は `design-reference/MokusoApp.dc.html`（トーンは akatsuki を採用。色値は [03_デザイントークン.md](03_デザイントークン.md)）。

## 1. ページ全体

- 1ページ完結・モバイルファースト（設計幅 402px）。コンテナ `max-width: 460px; margin-inline: auto`
- 大画面でも中央一列のまま（レイアウトバリエーションを増やさない）
- フォント: Noto Serif JP（本文）/ Zen Old Mincho（「黙走会」ロゴ・スタンプ絵柄の「走」）/
  Klee One（押印ボタンのラベル専用。2026-07-10 オーナー選定）
- `prefers-reduced-motion: reduce` で明転・点滅アニメーションを無効化
- Cookie・分析タグ・外部リクエストなし（例外は Google Fonts のみ。[04_検証チェックリスト.md](04_検証チェックリスト.md) §6.5）

## 3. 印（#stamp）— 未押印 ⇄ 押印済み

リード文（常時）: **「走っても、歩いても、今日の印を。」**（12.5px / letter-spacing .26em。
2026-07-09 変更: 応援の声色を消し、憲章2条「歩いてもいい」を入口に置く — 西田ペルソナ対応）

### 3.1 未押印状態: 印ボタン + スタンプカード

2026-07-10 全面改稿（オーナー承認）: 御朱印の印影から「夏の朝のスタンプカード」へ。
キーワードは「手触り感の積み重ね・手書き感・毎日押したくなる」。

**印ボタン（判）**
- 168px 円形ボタン・朱のベタ塗り（グラデーション・光沢なし。ゴム印の澄んだ朱）
- ラベル: **「走った印」「を押す」** の横二行。Klee One 32px / 600 / letter-spacing .05em /
  `font-feature-settings: 'palt'`。ラベル全体を `rotate(-2.5deg)`（右肩上がり）、
  各行を `scale(.9, 1.09)`（縦長の字形）— 手書きの自然さの3点セット
- 下に **「一日一回まで」** 10px（Klee One・同じ傾き）
- `:active` で `translateY(4px) scale(.97,.95)` + 影が締まる（--btnShadowPress）— 判を押し込む感触
- box-shadow: `var(--btnShadow)`（ダークでは真っ黒の大きな影を使わない）

**スタンプカード（台紙）— 押印前から見えている**
- 316px（max-width 86%）・角丸10px・--paper 地・上辺中央に紐穴・淡い影（--cardShadow）
- 10マス（5×2）。走った日の分だけスタンプが並ぶ（未押印マスは細線の空円）
- スタンプ絵柄は4種が日替わりで混在: **走の印**（丸枠+走・Zen Old Mincho）/ **花丸** / **朝日** / **水面**（一粒の月と波紋）
- n日目の印はシード `minamo-stamp/n` で決定（絵柄・角度±6.5°・位置±2px・濃さ）。
  **一度押された印は翌日以降も同じ姿**（粒と同じ「動かない過去」/ 07参照）。かすれ・グランジ加工はしない
- 満了したカードは下に重なる（1枚=4pxの縁、最大10段）+ **「いっぱいになったカードが、下に {n} 枚」**
- 日付マスを持たないため、休んだ日が空白として残らない（最重要原則5）

### 3.2 押印の瞬間 — 所作の三拍

1. localStorage 更新（§8）
2. **明転**: 画面全体に約1.25秒の夜明け色オーバーレイ（従来どおり mkFlash）
3. **所作**: 押し込む（:active）→ 判が持ち上がって退場（mkPressLift .5s / press-seq クラス）→
   430ms 後にカードへ今日の印が **ぽん** と着地（mkPon、カードは mkDip で受け止める / pressed-now クラス）
   → 添え書きエディタを静かに提示。所作は押したセッションだけ動き、リロードでは再生しない
4. サーバへ `apiStamp()`（[02_設計図_API_データ.md](02_設計図_API_データ.md)。失敗しても無言 — 層1は既に成立している）

### 3.3 押印済み状態

- ボタンは退場し、カードの最後に今日の印が灯っている（.today = 濃さ .95）
- カードの下に **「これで {total} 日目です」**（total は 32px / --ink。朱の規律: 個人の数字は墨）
- さらに下: **「今日は、もう印を残しています」** / **「また、あした。」**（--sub。
  2026-07-09 変更: 「お待ちしています」は運営の声のため削除。
  2026-07-10 変更: 「あしたの朝に」は利用時間帯を想定するため「また、あした。」へ — 走る時刻は各自のもの）

ボタンに戻る手段はない（取消・連打の概念を持たない。一回性を挙動で表現する）。

## 4. 添え書き（#note）— 3態

押印済みのときのみ、いずれか1つを表示する。

### 4.1 エディタ（押印直後に自動表示 / 「書き直す」「添え書きする」で再表示）

- 見出し: **「添え書きしますか?」** + 小さく **「(任意)」**
- 入力2つ（横並び・flex:1）: placeholder **「距離(km)」** / **「時間(分)」**
  - `<input type="text" inputmode="decimal">`（数字キーパッド直起動）・フォントサイズ17px・padding 13px（誤タップ防止）
- ボタン2つ（横並び・flex:1・タップ高さ約46px）: **「添える」**（--accent地・白文字）/ **「あとで」**（枠線のみ）
- 案内文: **「入力は任意です。空欄のままでもかまいません。」**
- ファイル導線（破線枠のlabel・中にinput[type=file]非表示）: **「記録ファイルを添える(GPX / TCX / FIT)」**
  `accept=".gpx,.tcx,.fit"`。ドラッグ&ドロップも受ける（§7）
- 解析結果メッセージ表示欄（成功時は入力欄への転記、失敗時は§7の1行のみ）

保存ロジック: km・分とも空/0以下なら何も保存せず閉じる。書き直し時は sumKm から旧kmを引いてから新kmを足す。

### 4.2 サマリー（当日の添え書きがあるとき）

**「今日の添え書き:{km} km ・ {分} 分」** + 下線リンク **「書き直す」**
（km/分は入力があった方だけを「 ・ 」区切りで表示）

### 4.3 リンク（押印済み・添え書きなし・エディタ非表示のとき）

下線リンク **「添え書きする(任意)」** のみ。

## 5. 三層カード

共通: --paper 地・1px --line 枠・radius 8px・padding 20px 22px。
ヘッダー行: 番号（--accent・14px）+ タイトル（14px）+ 右端に属性ラベル（10px・--sub）。下線区切り。

### #sec01 わたしの積み重ね（非公開）

- total > 0: **「{total} 日」**（38px）+ **「総走行日数(累計のみ)」**
  - sumKm > 0 のときだけ: **「添え書きの累計:{sumKm} km」**
- total = 0: **「まだ、印はありません。最初の一歩から始まります。」**
- 脚注（常時): **「あなたにだけ見える、静かな累積です。休んでも、減りません。」**

### #sec02 今日の静かなつながり（匿名）

- **「今日、共に走った人 {count} 人」**（count は 38px / --accent）
- 脚注: **「誰のものでもない数字です。明日には、またゼロから始まります。」**
- count はサーバの `GET /api/today` 由来（5分キャッシュ・目安の数字でよい）。
  自分が押印した直後、手元の表示が古い場合は +1 して見せてよい（サーバ側には既に自分が含まれている）
- **API不達時は数字を「—」にする**（説明文はそのまま。エラー表示・リトライボタン・スピナーは置かない）

### #sec03 会釈（匿名・一方向）— ダークカード

- 背景 --night・文字 --nightInk。ヘッダー番号は --glowInk
- **受信バナー**（自分宛の会釈が存在するときのみ・ヘッダー直下）:
  小さな光点（9px・#f4d9a8・mkPulse 3s）+ **「今日のあなたの走りに、誰かが会釈しました。」**
  受信は1ビット。数・履歴・送り主は存在しない
- **光の点フィールド**（height 188px・position:relative）:
  - 表示点数 = `min(38, max(6, count))`
  - 配置は日付文字列シードの疑似乱数（FNV-1aハッシュ → mulberry32。design-reference の `rng()` を移植）で
    中央寄りに散らす（left 5〜94% / top 10〜86%）。同じ日は誰が見ても同じ星空になる
  - 点: 3〜6.5px の円・#f2e3c4・微光 box-shadow・`mkTwinkle 2.4〜5.4s` の明滅
  - 未押印時は opacity .14（気配だけ見える）。押印済みで opacity 0.3〜0.9
    （触れられる点は下限 .55 に持ち上げる。乱数の消費順は変えない）
  - **タップ領域は不可視の44×44px**（点は小さくても押しやすく）
  - 匿名ID（自分のIDは除外）は**巡回割り当て** `serverDots[i % length]` — どの光に触れても実在する
    だれかに届く。「触れたのに無反応」の点を作らない（2026-07-10 UX改善。点は個人に紐づく表示では
    ないため巡回でよい。IDがゼロ件のときだけ全点タップ不可）
- **会釈の送信**: 押印済み・未会釈のときだけ点がタップ可能。触れられる点には**淡い輪**
  （::after 26px円・1px rgba(242,227,196,.3)）が現れ、`:active` で点が 1.45倍にふわっと応える。
  タップで `POST /api/bow`。会釈した点は --glowInk に灯り `mkPulse 2.6s`、リング状の box-shadow を持つ
- **状態別の言葉（2026-07-10 改稿）**: ガイド行 #bow-lead（フィールド上・12px）が「いま何ができるか」を
  先に言い、キャプション #bow-caption（フィールド下・10.5px）が作法を添える:
  1. 未押印: **「光の一粒は、今日すでに走っただれかの気配です。」** / **「走った印を残すと、ひとつの光に会釈できます。」**
  2. 押印済み・未会釈・光あり: **「光をひとつ選んで、そっと触れる。それが今日の会釈です。」** / **「一日に一度だけ。誰か分からない、返せない会釈です。」**
  3. 押印済み・未会釈・光なし（他の参加者ゼロ）: **「今日は、まだほかの走りの気配がありません。」** / **「だれかが印を残すと、ここに光が灯ります。」**
  4. 会釈済み: **「そっと、会釈を残しました。」** / **「相手には、灯りがひとつだけ届きます。だれからかは、伝わりません。」**
- **API不達時はカードごと display:none**（静かに畳む）

## 7. ファイル取り込み（Step 3 実装）

- 解析は**100%クライアントサイド**。ハンドラ内に fetch / XHR / sendBeacon を一切書かない
- 抽出するのは**距離(km)・時間(分)の2値のみ**。GPSルート・心拍・ケイデンス等は変数にも取り出さない
- 結果は添え書き入力欄への**プリフィルのみ**（自動保存しない。「添える」でユーザーが確定する）

| 形式 | 方法 | 抽出 |
|---|---|---|
| GPX | `file.text()` → DOMParser | 距離: trkpt の lat/lon をハバーサイン法で積算。時間: 最初と最後の `<time>` の差 |
| TCX | 同上 | 距離: Lap/DistanceMeters 合計（なければ最後の Trackpoint/DistanceMeters）。時間: Lap/TotalTimeSeconds 合計 |
| FIT | `file.arrayBuffer()` → `js/fit-reader.js` を遅延読み込み | session メッセージ（global msg 18）の total_distance(field 9)・total_timer_time(field 8) のみ |

- FITパーサは**CDN読み込み禁止**。第一候補は自作最小リーダー（~200行）。難航時は MIT の `fit-file-parser` を
  リポジトリに同梱（vendoring・ライセンス表記）
- 読み込みは**初回の .fit 選択時に classic script を遅延注入**する（当初案の動的importは、憲章7条の
  「フォルダをコピーして file:// で直接開く複製」で CORS ブロックされるため変更。2026-07-09）。
  意図は同じ = **使わない訪問者には1バイトも読ませない**
- 解析失敗時の文言（1種類のみ・静かに）: **「このファイルは読めませんでした。手で書き添えることもできます。」**

### URLフラグメント受け口（iOSショートカット用）

- ページロード時に `location.hash` を確認。`#d=5.2&t=31`（d=km, t=分）なら添え書き欄にプリフィルし、
  直後に `history.replaceState(null, '', location.pathname + location.search)` でURLから即消去
- フラグメントはサーバに到達しないため、記録データはネットワークに出ない

## 8. クライアント状態（localStorage）

キー名は名前空間 `minamo.` に統一（プロトタイプの `mokuso.{tone}.` から簡素化）:

| キー | 値 | 用途 |
|---|---|---|
| `minamo.total` | 数値 | 累計日数（唯一の積み重ね。**連続性の計算には使わない**） |
| `minamo.last` | `"YYYY-MM-DD"` | 同日重複押下の判定**のみ**に使用 |
| `minamo.note` | `{"date","km","min"}` | 当日の添え書き（翌日は自然に「当日分なし」になる） |
| `minamo.sumKm` | 数値 | 添え書きの累計km（小数1桁丸め） |
| `minamo.anon` | `{"date","id"}` | その日のサーバ発行匿名ID |
| `minamo.bowed` | `{"date","idx"}` | 会釈済み日付 + どの点か（同日再訪時の描画用） |

- 日付キーはすべて JST: `new Intl.DateTimeFormat('en-CA', { timeZone: 'Asia/Tokyo' }).format(new Date())`
- **サーバに送るデータはゼロ**（層2/3のAPIはボディ空 or 匿名IDのみ。距離・時間は絶対に送らない）
- 押印ロジック: `last === today` なら何もしない → `total++` / `last = today` → 明転 → エディタ表示 → apiStamp()

## 9. グレースフルデグラデーション（実装必須）

| 状況 | 挙動 |
|---|---|
| `API_BASE = ""`（複製版・ローカル） | 層1のみで完全動作。#sec02 は「—」、#sec03 は非表示 |
| fetch 失敗 / 4秒タイムアウト（AbortController） | 同上。コンソールにエラーを撒き散らさない |
| 押印時のサーバ同期失敗 | 無言で受け流す。次回ロード時に `last === today && anon の date ≠ today` なら1回だけ静かに再送 |
| localStorage 不可（プライベートモード等） | try/catch で握り、その場のメモリ動作のみ（ページは壊さない） |

file:// で直接開いても層1が成立すること。
