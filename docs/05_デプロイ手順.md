# 05_デプロイ手順 — Cloudflare Pages / GitHub Pages 両対応

ホスティングは **Step 4 で決定**する（[00_計画書.md](00_計画書.md) §2）。バックエンド（Workers + KV）はどちらを選んでも共通。
当面は無料サブドメインで公開し、minamo.run の接続は後日（§4）。

---

## 1. 共通: Workers + KV（Step 2）

前提: Cloudflareアカウント（無料プラン）、Node.js、`npm i -g wrangler`（またはnpx）。

```bash
cd worker
wrangler login                              # ブラウザ認証
wrangler kv namespace create MINAMO_KV       # 出力された id を wrangler.toml に貼る
wrangler deploy                              # → https://minamo-api.<account>.workers.dev
```

動作確認（[04_検証チェックリスト.md](04_検証チェックリスト.md) §2〜3 の curl 一巡）:

```bash
API=https://minamo-api.<account>.workers.dev
curl -X POST $API/api/stamp                  # → {"date":"...","anonId":"..."}
curl $API/api/today                          # → {"date":"...","count":1,"dots":["..."]}
curl -X POST $API/api/bow -H "Content-Type: application/json" -d '{"to":"<anonId>"}'
curl "$API/api/bow?date=<date>&id=<anonId>"  # → {"bowed":true}
```

デプロイ後、`public/index.html` の `API_BASE` にこのURLを設定する。

**注意:** wrangler.toml に `[observability]` を追加しない（ログ収集を有効化しない）。
ダッシュボードの Workers Logs / Analytics Engine も無効のままにする（[02_設計図_API_データ.md](02_設計図_API_データ.md) §6）。

**タイムゾーン（複製時）:** `worker/src/index.js` の `TIME_ZONE` 定数は `public/js/minamo-core.js` の
同名定数と同じ値にする。ズレると層1と層2/3で日の変わる瞬間が分かれる（データ喪失はなく、TTL で
消える範囲の静かなズレに留まる）。

## 2. 案A: Cloudflare Pages（Workersと同一プラットフォーム）

### 2-1. GitHub連携（推奨・push で自動デプロイ）

1. GitHubにリポジトリを公開（Step 4）
2. Cloudflare ダッシュボード → Workers & Pages → Create → Pages → Connect to Git
3. 設定: Build command **なし** / Build output directory **`public`**（ビルド不要の純静的サイト）
4. デプロイ完了 → `https://<project>.pages.dev`

### 2-2. CLI直デプロイ（Git連携なしで済ませる場合）

```bash
wrangler pages project create minamo
wrangler pages deploy public --project-name minamo
```

### 2-3. 仕上げ

- `worker/src/index.js` の `ALLOWED_ORIGINS` に `https://<project>.pages.dev` を設定して `wrangler deploy`
- [04_検証チェックリスト.md](04_検証チェックリスト.md) §8 を通す

## 3. 案B: GitHub Pages

`docs/` フォルダはドキュメントで使用しているため、**Actions ワークフローで `public/` を配信**する方式を取る。

### 3-1. `.github/workflows/pages.yml`

```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
permissions:
  contents: read
  pages: write
  id-token: write
jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/upload-pages-artifact@v3
        with:
          path: public
      - id: deployment
        uses: actions/deploy-pages@v4
```

### 3-2. 手順

1. リポジトリ Settings → Pages → Source を **GitHub Actions** に設定
2. main へ push → `https://<user>.github.io/<repo>/`
3. `ALLOWED_ORIGINS` に `https://<user>.github.io` を追加して `wrangler deploy`

**注意:** プロジェクトページはサブパス（`/<repo>/`）配下になる。index.html 内のリンク・fit-reader.js の
参照は相対パス（`./js/fit-reader.js`）で書くこと（案Aとの両対応のため、絶対パス `/js/...` を使わない）。

## 4. minamo.run 接続（後日・手順のみ）

1. **取得**: 任意のレジストラで minamo.run を取得（.run TLD）。Cloudflare Registrar を使うなら
   先にCloudflareへゾーン追加が必要
2. **DNSをCloudflareに向ける**（案A/Workersと一元管理するのが自然）: Cloudflareにサイト追加 →
   レジストラでネームサーバをCloudflare指定値に変更
3. **静的ページ**:
   - 案A: Pages プロジェクト → Custom domains → `minamo.run` を追加（CNAMEは自動設定）
   - 案B: リポジトリ Settings → Pages → Custom domain に `minamo.run` → `public/CNAME`（内容 `minamo.run`）を
     追加 → DNSに CNAME `<user>.github.io` を設定
4. **API**: workers.dev のままでよい（コード変更不要）。揃えたければ Workers → Domains & Routes で
   `api.minamo.run` を追加し、index.html の API_BASE と ALLOWED_ORIGINS を更新
5. 接続後、`ALLOWED_ORIGINS` に `https://minamo.run` を追加して再デプロイ

## 5. ローカル開発

```bash
# 静的ページ（どちらでも）
npx wrangler pages dev public          # http://localhost:8788
# または VS Code Live Server 等。file:// 直開きでも層1は動く

# Worker
cd worker && wrangler dev              # http://localhost:8787
# ローカル確認時は index.html の API_BASE を http://localhost:8787 に一時変更
# （ALLOWED_ORIGINS に localhost:8788 が入っているのでCORSは通る）
```

## 6. 運用（存在しないことの確認）

デプロイ後にやることは**ない**。日次リセット・バッチ・監視当番・KV掃除はすべて設計で不要化されている
（日付キー + TTL）。Cloudflareの無料枠通知メールが来たら [02_設計図_API_データ.md](02_設計図_API_データ.md) §3 の
フォールバック検討タイミング。
