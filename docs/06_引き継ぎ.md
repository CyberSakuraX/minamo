## 3. 実装対象（サマリー)

- `public/index.html` — 1枚完結の静的ページ（CSS/JSインライン）。層1はlocalStorageのみで完全動作
- `public/js/fit-reader.js` — FIT最小パーサ（動的import・Step 3）
- `worker/src/index.js` + `worker/wrangler.toml` — API 4本（stamp / today / bow送信 / bow確認）+ KV 2キー種 + TTL 36h
- `LICENSE`(MIT) / `README.md` / `.gitignore` / git init（Step 1 と 4）

プロトタイプに**なくて**本実装で足すもの: 憲章全文セクション・フッター・ファイル解析の実体
（プロトタイプのファイル取り込みは受領表示だけのモック）。

プロトタイプに**あって**本実装で捨てるもの: トーン切替（暁固定）・Tweaksパネル・プリセット機能
（`applyPreset` はデモ用）・`participants` プロップ（実装ではサーバの実数）。

## 4. 絶対禁止チェックリスト（コードを書く前に暗記する）

- [ ] ストリーク・連続日数・「途切れ」— **概念ごと禁止**（累計のみ）
- [ ] 会釈の数の表示・蓄積・履歴 — 受信は1ビットのみ
- [ ] 他者から見える個人紐づきの数字（距離・時間・ペースをサーバに送らない/他者に見せない）
- [ ] ログイン・アカウント・Cookie・分析タグ・トラッキング
- [ ] cron・定期実行・リセット処理・KV delete（消すのはTTLだけ）
- [ ] DM・コメント・返信・連絡手段
- [ ] IP・UAの保存（レート制限のメモリ内一時利用のみ可）
- [ ] 過剰なabuse対策（CAPTCHA等）— 簡易レート制限まで
- [ ] 保留箱の機能（一筆・Strava/Garmin公式API連携）— 実装も提案もしない

機能を足したくなったら PERSONA.md の4つの問い（藤井は静かに喜ぶか / 小川は参加できるか /
西田は怖がらないか / アンチペルソナは飢えたままか）。**ひとつでもNoなら作らない。迷ったら削る。**

## 6. つまずきそうな点（計画セッションからの申し送り）

1. **KVに原子的インクリメントはない** — 層2は `runner:{date}:{anonId}` の独立キー + list カウント方式
   （[02_設計図_API_データ.md](02_設計図_API_データ.md) §3 に採用理由とフォールバック案）。カウンタキーのRMWを書かないこと
2. **日付はサーバ・クライアントとも同一のJST式** — `Intl.DateTimeFormat('en-CA', {timeZone:'Asia/Tokyo'})`。
   `toISOString().slice(0,10)` を使うとUTC日付になり9時間ズレるバグになる
3. **workers.dev では Cache API が使えない** — GET /api/today のキャッシュはモジュールスコープ変数 +
   Cache-Controlヘッダの2段（[02_設計図_API_データ.md](02_設計図_API_データ.md) §4）
4. **明転は1回きり・1.25秒** — forwards で終わらせJSで要素除去。ループさせない
5. **点のタップ領域は44×44px**、見た目の点は3〜6.5px — 分けて実装する（原本の wrap / dot 構造）
6. **添え書きの書き直し** — sumKm から旧kmを引いてから新kmを足す（原本 `saveNote` 参照）
7. **自分の点** — GET /api/today の dots から自分の anonId を除外して描画（自分に会釈できてしまう）
8. **fit-reader.js は相対パス参照**（`./js/fit-reader.js`）— GitHub Pages のサブパス配下でも動くように
9. **エラーを飾らない** — 失敗はすべて「静かに畳む」（[01_設計図_画面と挙動.md](01_設計図_画面と挙動.md) §9）。
   エラートースト・リトライUI・スピナーは設計違反
