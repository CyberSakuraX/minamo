## 1. 何であるか

層1「わたしの積み重ね」の累計日数を、数字とは別に**一枚の風景**として描く。
ヘッダー下にもともとある丘のシルエット+朝日のグローが、積み重ねに応じて静かに育っていく。
新しいセクションは足さない。**既にある風景が変わる。**

出典: NAMING.md「積み重ねは『風景の複利』として可視化（粒→稜線→月→水面→夜明け）」。

## 2. 風景の文法（憲法。絵柄より先にこれを守る）

1. **入力は `minamo.total` ただひとつ。** 日付・連続性・頻度は一切見ない
2. **決して後退しない。** 風景は増えるだけ。休んでも一粒も消えない
   （「休んでも、減りません」が言葉ではなく絵になる）
3. **自分にだけ見える。** サーバーに送らない・他者の風景は存在しない・比較不能
4. **静止画である。** 常時動く装飾は層3の点の明滅のみ（03の原則）。風景にアニメーションを足さない
5. **通知しない。** マイルストーン到達を知らせるバナー・お祝い演出は作らない。
   ある朝ふと気づく、が正しい出会い方

## 3. マイルストーン（叩き台 — 日数はオーナーが決める）

| 累計 | 風景 | 意味 |
|---|---|---|
| 1日〜 | **一粒の光**が丘に灯る。以後1日ごとに一粒ずつ増える（上限24粒） | 最初の一歩 |
| 30日〜 | **稜線**が定まる（ぼやけていた丘の輪郭に細い線が現れる） | 続いてきた道のかたち |
| 90日〜 | **月**が昇る（空に小さな一粒の月。ロゴ意匠と同じモチーフ） | 夜を走る人の証人 |
| 240日〜 | **水面**が現れ、光を映す（丘の麓に細い光の帯） | **minamo** — 名前の風景に到達する日 |
| 365日〜 | **夜明け**が差す（空に淡い暁色、朝日のグローが強まる） | 一年分の朝 |

- 240日=水面 は NAMING.md の記述を踏襲。**名前の由来と風景が同じ場所で合流する**設計
- 粒の配置は固定シードの決定的乱数（`rng('minamo-grains')`）。
  n粒目の位置は永久に同じ=**昨日の粒は今日も同じ場所にある**（乱数は1粒あたり定数回消費）
- **粒と月は暁の灯色（`--glow-ink #eab88a` + 微グロー）**。明るい和紙の上での知覚は
  輝度差10〜14% + 暖色の色相差 + グローの三点で確保する（淡色 #f2e3c4 は夜空カードでは効くが
  和紙背景では輝度差1〜3%で消える — 2026-07-08 デザインレビューで検出・修正済み）。
  もっと存在感が欲しければ粒の色を `--accent-hi` に一行変更（輝度差約18%になる。オーナー判断）。
  月の位置は右上の空（left 78% / top 2px）— リード文と重ねない

## 4. 印影の日ごとのゆらぎ（同梱の小さな提案）

- 押印後の印影に、日付シード（`rng('minamo-seal/' + YYYY-MM-DD)`）の決定的なゆらぎを与える:
  回転角 -2.5°〜-4.5° / 内リングの余白 7〜10px / 印の濃さ 0.94〜1.00
  （濃さの下限は0.94。それ以下は「印が薄れた」= 劣化の記号として読まれ得るため。ゆらぎの主役は回転と内リング）
- **同じ日は誰の印影も同じゆらぎ**（層3の星空と同じ「共有される気配」）
- 落款の日付は従来どおり。データは何も増えない

## 5. やらないこと（この機能に関する再確認）

- **御朱印帳ビュー（過去の印の一覧）** — カレンダー型は押さなかった日が空白として見え、
  「途切れ」を責める構造が復活するため作らない。積み重ねの可視化は風景に一本化する
- 他者の風景を見る機能 / 風景の共有・書き出しボタン / マイルストーン到達通知 /
  風景のカスタマイズ（テーマ・色選択）/ 過去日付の遡り押印

## 8. オーナーに決めてほしいこと

1. マイルストーンの日数（30/90/240/365 のままでよいか。240=水面 は固定を推奨）
2. 粒の上限（叩き台は24。増やすと賑やか、減らすと静か）
3. 月と水面の絵柄の趣（モックを見て「もっと薄く/小さく」等の一言でよい）
4. 印影ゆらぎの採否（風景と独立に採否を決められる）
