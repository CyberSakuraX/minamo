## 採用

### バックアップ / 復元（JSON・端末内完結）

- 「休んでも、減りません」の約束を、端末故障・OSクリア・機種変更から守るのは思想と整合する
- Web で完結（サーバー送信なし・ネットワーク不要）
- UI は #sec01 の footnote 直下に控えめなテキストリンク 2 つ。書き出しは積み重ねがあるときだけ
- 復元は全面上書き（ファイルが真実）。適用前にインライン確認（`confirm()` は使わない）
- 対象キー: total / last / sumKm / note。anon / anonPrev / bowed は「翌日に消える」思想のため含めない

### 最小自動テスト（`node --test`・依存ゼロ）

- 日付境界・状態遷移・Worker API・会釈引き継ぎ・バックアップ往復の 5 領域
- CI・ビルド工程・npm 依存はゼロのまま
- `worker/package.json` は `{"type":"module"}` の 1 行のみ（**リポジトリ root には絶対に置かない** —
  `public/js/minamo-core.js` が ESM 判定されて壊れる）

### TIME_ZONE の定数化

- 憲章 7 条「どこの街でも自由にひらいてよい」との整合。ハードコードは複製者に不親切だった
- クライアント側は `public/js/minamo-core.js`、サーバー側は `worker/src/index.js` の同名定数
- **両者を同じ値にする**必要がある。README・docs/05 に手順を追記
- ズレたときの挙動: 層1 と層2/3 で日の変わる瞬間が分かれる。データ喪失はなく、TTL で消える範囲の
  静かなズレに留まる（走った気配は正しく残り、明日には片付く）

### anonPrev 掃除バグ修正（レビュー副産物）

- `localStorage.removeItem('anonPrev')` がプレフィックスなしで呼ばれており、実キー
  `minamo.anonPrev` が永久に残っていた（機能影響なし・掃除の空振り）
- `store.remove('anonPrev')` に置換し、`test/state.test.mjs` に「実キーが消えること」の固定を追加

## クライアント/Worker の TIME_ZONE がズレたときの挙動

TIME_ZONE を複製者が変えるとき、うっかり片側だけ変えても壊滅はしない。想定される差異:

- クライアント（layer 1）が判定する「今日」と、Worker（layer 2/3）が判定する「今日」が数時間ズレる
- 層1: 押印は端末側の「今日」で加算される。同日重複ガードは端末側で完結するため、二重加算は起きない
- 層2: `runner:{date}:{anonId}` のキーがクライアント側 date と Worker 側 date で不一致になる可能性。
  ただし全キーに TTL 36h が付いており、翌日には片付く。混ざるとしても人数カウントに±1程度の静かなズレ
- 層3: 会釈の送信/照会も日付キー基準。ズレた場合、送信当日中に受け取れないだけで、TTL 内に消える

**データ喪失は起きない。TTL で消える範囲の静かなズレに留まる。**
複製者にとって「一度合わせ忘れても壊れない」設計は、憲章 7 条の「自由にひらいてよい」を裏で支える。
