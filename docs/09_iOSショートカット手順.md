# 09_iOSショートカット手順 — ヘルスケアから静かに添え書き

iOSの「ショートカット」アプリで、直近のランニングの距離と時間を minamo に一発でプリフィルするレシピです。

- **通信は発生しません。** 数値はブラウザのURLフラグメント(`#d=<km>&t=<分>`)として渡され、
  サーバーには一切届きません(HTTPリクエストに`#`以降は含まれない仕様)
- 受け取ったフラグメントは、ページ側で読み取った直後にURLから消えます([public/index.html](../public/index.html) の `applyFragment`)
- 使わない人は、これまで通り手入力で完全に成立します。あくまで **Appleヘルスケアと二重入力したくない人向けの控えめな導線**です

## 前提

- iOS 17以降(古いバージョンでも大筋は動きますが、アクション名の日本語ラベルが多少ずれます)
- ヘルスケアにランニング(または歩きの)ワークアウトが記録されている
  - Apple Watch / Nike Run Club / Strava / 各種自動記録アプリのいずれか
- ブラウザで minamo をブックマーク済み(現時点: https://minamo-eyu.pages.dev / minamo.run 接続後は差し替え)

## レシピ(9ステップ)

「ショートカット」アプリ → 右上「+」→ 以下を上から順に置いていきます。

| # | アクション | 設定 |
|---|---|---|
| 1 | **ワークアウトを検索** | フィルタ: `ワークアウトのタイプ = ランニング` / 並べ替え: `終了日` 最新順 / 制限: `1` |
| 2 | **詳細を取得** | 対象: ↑のワークアウト / 取得: `総距離` |
| 3 | **単位を変換** | 変換先: `キロメートル` |
| 4 | **数字を丸める** | 小数点以下 `1` 桁 → 変数名を `km` に |
| 5 | **詳細を取得** | 対象: ステップ1のワークアウト / 取得: `継続時間` |
| 6 | **計算** | ↑ ÷ `60` (秒 → 分) |
| 7 | **数字を丸める** | 小数点以下 `0` 桁 → 変数名を `min` に |
| 8 | **テキスト** | 内容: `https://minamo-eyu.pages.dev/#d=[km]&t=[min]` (角括弧は変数として挿入) |
| 9 | **URLを開く** | 対象: ↑のテキスト |

初回実行時に「ワークアウトの読み取り」の許可ダイアログが出るので、そのまま許可します。

## 置き場所

好みで一つだけ選べば十分です(足しすぎると走り終えた直後の視界がうるさくなります)。

- **ホーム画面**: 詳細画面 → 「ホーム画面に追加」
- **ウィジェット**: ホーム画面長押し → ショートカット
- **Apple Watch**: Watch アプリ → マイウォッチ → ショートカット

## 動作確認

1. 走り終える。ヘルスケアにワークアウトが乗る
2. ショートカット実行 → ブラウザに `https://.../#d=5.2&t=31` のような形で開く
3. 一瞬でURLから `#...` が消える(端末に痕跡を残さない)
4. 押印済みなら添え書き欄が展開されて数値が入る。未押印なら、押印を待って静かに流し込まれる

## 動かないとき

- **権限が拒否済み**: 「設定 → プライバシーとセキュリティ → ヘルスケア → ショートカット」で読み取りを許可
- **ワークアウトが見つからない**: ステップ1のフィルタ「ランニング」を一旦外して、直近のワークアウトが取れるか確認
- **URLが正しく開かない**: ステップ8で `#` と `&` が全角になっていないか確認
- **距離が0.0になる**: 単位変換(ステップ3)の変換元が距離になっているか、二重変換していないか確認

## 補足

歩いた日を添え書きしたいときは、ステップ1のフィルタを `ウォーキング` に変えたショートカットを別建てで作ると混ざりません。ただ、憲章の精神から言えば、**添え書きは任意**です。走った(歩いた)印を押した、それだけで参加は完全に成立します。
